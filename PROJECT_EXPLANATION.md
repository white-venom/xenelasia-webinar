# Xenelasia Webinar Management & Registration Portal — Project Explanation

This document explains the entire project architecture, the tools and technologies used, and what specific files/libraries are used for what features.

---

## 1. Project Architecture Overview

The system is structured as a **Monorepo** consisting of two decoupled layers:
1. **Frontend (`/frontend`)**: A client-facing Next.js application designed with futuristic, responsive layout design, glassmorphism cards, and interactive 3D particle graphics.
2. **Backend (`/backend`)**: A RESTful Express.js API server built using TypeScript and structured with secure authentication, automated registration passes, completion certificates, and chatbot simulation endpoints.

```
Webinar (Root Workspace)
├── backend/                  # REST API Server (TypeScript + Prisma ORM + Express)
│   ├── prisma/               # Database Schema and Seeding Configuration
│   └── src/                  # Route Handlers, Database Configurations, Middlewares
├── frontend/                 # Client UI (Next.js App Router + Tailwind CSS + Three.js)
│   ├── src/app/              # Next.js Pages (Routes) and Layouts
│   ├── src/components/       # Shared UI Components (Navbar, 3D Canvas, Chatbot)
│   └── src/utils/            # HTTP Request wrappers and API Integrations
├── package.json              # Monorepo Orchestration (Start/Build concurrent scripts)
└── .env                      # Global Environment Settings (Port, Secrets, DB location)
```

## 2. Technology Stack: What We Use For What

Here is a comprehensive breakdown of every major technology, framework, and package used in this project and why they were chosen:

### 2.1 Core Frameworks & Runtime
*   **Node.js**: The runtime environment executing JavaScript/TypeScript code on both the server and client build environments.
*   **Next.js (v16)**: The React framework powering the frontend.
    *   *Used for:* Routing via the Next.js App Router, handling SSR (Server-Side Rendering) for static layouts, and providing client-side hydration for dynamic page features.
*   **Express.js**: The routing and endpoint engine on the backend.
    *   *Used for:* Defining API endpoints (`/api/auth`, `/api/webinars`, etc.), handling CORS preflight configuration, parsing request payloads, and executing business logic.
*   **TypeScript**: A typed superset of JavaScript.
    *   *Used for:* Bringing static type checks to both frontend and backend directories, catching syntax and object mapping errors before compilation.

### 2.2 Database & Data Management
*   **Prisma ORM**: A modern object-relational mapper for TypeScript.
    *   *Used for:* Translating SQL database tables into typed TypeScript objects. Prisma is used to write queries (e.g. `prisma.webinar.findMany(...)`) instead of raw SQL queries.
*   **SQLite**: A lightweight, file-based SQL database engine.
    *   *Used for:* Storing all portal registrations, cybersecurity blogs, certificates, notifications, and passwords locally in a single file (`/backend/prisma/dev.db`) without requiring external database installations (like Docker or cloud Postgres clusters).
*   **Prisma Client**: Autogenerated wrapper library.
    *   *Used for:* Direct programmatic access to databases with auto-completion inside code editors.

### 2.3 Styling, UI, & Animations
*   **Tailwind CSS (v4)**: A utility-first CSS framework.
    *   *Used for:* Structuring layout systems, dark-themed styling (slate-950), sizing, spacing, glassmorphic card overlays, responsive designs (mobile vs desktop), and custom neon typography glows.
*   **Three.js**: A cross-browser 3D JavaScript library.
    *   *Used for:* Rendering the premium custom particle network element (`ThreeBackground.tsx`) running at 60 FPS behind the text. It computes particle connections, node repulsion during mouse movements, and automatic scene rotation.
*   **Framer Motion**: An animation library for React.
    *   *Used for:* Smooth scale and position transformations when pages load, modal entries, chat window slide-ups, and interactive button hover/click micro-animations.
*   **Lucide React**: Vector icon pack library.
    *   *Used for:* Displaying crisp symbols throughout the UI (such as security shields, key locks, checkmarks, download buttons, calendar grids, and user avatars).

### 2.4 Security & Authentication
*   **JSON Web Tokens (JWT)**: Security token standard.
    *   *Used for:* Generating signed tokens (`jwt.sign()`) upon user login. The token is stored in the browser's `localStorage` and sent in the `Authorization: Bearer <token>` header of every subsequent API call to verify the user's identity securely.
*   **bcryptjs**: Password hashing library.
    *   *Used for:* Salting and hashing passwords (`bcrypt.hash()`) during user registration. This ensures that raw passwords are never saved in the database, protecting user accounts if the database is exported.
*   **Helmet**: Security middleware for Express.
    *   *Used for:* Securing backend endpoints by setting various HTTP response headers to protect against common web exploits (XSS, Clickjacking, MIME sniffing).
*   **Express Rate Limit**: Middleware.
    *   *Used for:* Limiting requests from identical IP addresses over short time intervals to prevent brute-force login attempts and DDoS service interruptions.
